# StudyLock — 双模式深度工作助手

> 基于 Cal Newport《深度工作》理论，专为大学生打造的「小而精」专注锁机 APP。

---

## 理念

### 为什么做这个 APP

大多数专注类 APP 追求「大而全」——番茄钟、白噪音、Todo List、统计报表、社交激励……功能堆砌得越来越多，越来越像一个臃肿的平台。这不是我想要的。我要的是一个**在最需要的时刻，简单粗暴地让手机变得不可用**的工具。仅此而已。

### 理论基础：Cal Newport《深度工作》

Cal Newport 在《深度工作》一书中提出了四种深度工作策略：

| 策略 | 描述 | 适合人群 |
|------|------|---------|
| 僧侣式 (Monastic) | 彻底消除浅薄工作，长时间与世隔绝 | 作家、理论研究者 |
| **双模式 (Bimodal)** | **把时间明确划分为深度时段和浅薄时段** | **大学生、知识工作者** |
| 节奏式 (Rhythmic) | 每天固定时间段做深度工作，形成链条 | 上班族、自由职业者 |
| 记者式 (Journalistic) | 在任何空闲缝隙中挤入深度工作 | 日程不固定的人 |

**StudyLock 选择了双模式**。因为对于大学生来说，这个划分天然存在：

- **上课时间** → 自动深度时段（手机自动锁定，被动纪律）
- **空闲时间** → 手动启动深度时段（自选 30/60/90/120 分钟，主动选择）

你不是靠意志力去抵抗手机的诱惑——你是在上课前就已经决定了「这段时间手机不可用」。系统替你执行这个决定。这就是 Newport 所说的**被动的纪律**。

### 设计原则

- **小而精，不是大而全**。不做 Todo List、不做白噪音、不做统计报表、不做社交。只做一件事：在你需要专注的时候锁住手机。
- **被动纪律 > 主动意志**。课表导入后，上课时间到了自动锁。你不需要「决定」要不要专注——系统替你做了这个决定。
- **不可绕过**。按 Home 键会自动弹回锁机页面。按返回键无效。从多任务界面划掉也没用。除非输入密码或使用紧急解锁。
- **白名单而非黑名单**。锁机期间只放行计算器、时钟、日历、词典等学习工具。不需要你手动配置什么 APP 该拦什么不该拦。
- **最少权限**。只要求 1 个必要权限（显示在上层）。不需要无障碍服务，不需要使用情况访问。开箱即用，不折腾。

### 为什么不是番茄钟

番茄钟（25 分钟专注 + 5 分钟休息）是一种很好的时间管理方法，但它有一个根本性问题：**你随时可以取消它**。当手机弹出「休息时间到了」的通知，你很容易顺势刷 10 分钟短视频，然后告诉自己「再来一个番茄钟」。

StudyLock 不做番茄钟。你设定的是连续深度工作时长（30/60/90/120 分钟），期间手机就是砖头。没有「休息提醒」，没有「完成奖励」。只有倒计时归零后你才能解锁。这才是真正的深度工作。

## 功能

### 双模式锁机

| 模式 | 触发方式 | 适用场景 |
|------|---------|---------|
| **自动模式** | 导入课表后，上课时间自动触发锁机 | 课堂专注 |
| **手动模式** | 主屏点击「开始深度工作」，选择时长 | 自习、图书馆、备考、深夜赶 due |

### 课表 OCR 导入

拍照或从相册选择课表图片，Google ML Kit 自动识别中文文字，解析出课程名称、时间、地点、周次。支持：
- 表格格式（大学常见的行列排版）
- 列表格式（教务系统截图）

识别后可以逐门课程编辑修正，支持手动添加遗漏或删除错误识别的课程。

### 白名单应用

锁机界面上有 4 个学习工具可以直接使用：

| 工具 | 说明 |
|------|------|
| 🔢 计算器 | 数学/工程计算 |
| ⏰ 时钟 | 查看时间、设置闹钟 |
| 📅 日历 | 查看日程 |
| 📖 词典 | 查单词 |

其他 APP 在锁机期间全部不可用。

### 紧急解锁

每天 **3 次**紧急解锁机会，每次 **5 分钟**。用于接电话、回紧急消息等场景。用完即止，不可累积。

### 密码保护

支持 BCrypt 加密的密码解锁。连续 3 次输错锁定 10 分钟。可选「冷静期」功能——密码正确后需要等 60 秒才能真正解锁，防止冲动。

### 连续专注链条

追踪你的深度工作连续天数——Cal Newport 引用的「不要断掉链条」原则。

## 权限说明

**必要权限（1 个）**：

| 权限 | 为什么需要 |
|------|-----------|
| 显示在其他应用上层 | 锁机页面需要覆盖在其它 APP 之上。这是唯一必须手动在系统设置中开启的权限。 |

**建议权限**：

| 权限 | 为什么建议 |
|------|-----------|
| 忽略电池优化 | 防止系统后台杀死守护服务，导致计时中断。 |

**不需要的权限**：

很多锁机/专注类 APP 会要求以下权限，StudyLock **不需要**：

- ❌ 无障碍服务 — 其他 APP 用它来拦截应用切换，StudyLock 直接全屏覆盖，不需要。
- ❌ 使用情况访问 — 不需要监控你在用什么 APP。
- ❌ 通知权限 — 可选，不影响锁机功能。

## 项目结构

```
app/src/main/java/com/studylock/app/
├── data/
│   ├── dao/                 # Room DAO 接口
│   ├── database/            # StudyLockDatabase + 数据迁移
│   ├── entity/              # Course, Semester, FocusRecord 等实体
│   └── repository/          # 仓库层
├── feature/
│   ├── deepwork/            # 深度工作主屏（大按钮 + 链条 + 时间线 + 权限面板）
│   ├── focus/               # 锁机页面（FocusBlockActivity）、密码设置、名言
│   ├── permission/          # 权限引导页 + 厂商保活指南
│   └── schedule/            # 课表展示、OCR 导入、课程编辑、节次配置
├── service/
│   ├── FocusGuardService    # 前台守护服务（计时 + 通知）
│   └── FocusTimeChecker     # 课表时间判断工具
└── ui/theme/                # Material 3 主题色彩
```

## 构建

```bash
git clone https://github.com/你的用户名/StudyLock.git
# 用 Android Studio 打开项目，Sync Gradle，Run 即可
```

或命令行：

```bash
./gradlew assembleDebug
# APK 输出在 app/build/outputs/apk/debug/
```
